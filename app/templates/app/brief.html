Requirements

You will be provided an incomplete web application. You are required to complete and test the web database application in Python. Describe the programming concepts and syntax used. Assume that an Agile Approach is to be used to develop the web application project and describe its use.

The application should be focused on a topic that is relevant to your job role and/or company. The application could be used to handle IT Assets, Contacts, Help Desk Tickets, or Customer Information. To suit the brief, you may need to simplify your real data. Ask your tutor for help if you are unsure.

The web application should feature a small database (minimum of two tables and maximum of four tables) that can be implemented as SQLite tables. For testing purposes, the tables in the final database should contain multiple sample records (rows). A variety of relevant data types should be included in table columns. Fields that can serve as the Primary and Foreign Keys should be included. If two or more tables are related, then an Entity Relationship Diagram (ERD) would be required to show the defined entities, their attributes, and relationships between the tables.

- Users should be able to explore records from all the database tables, as well as add, update, and delete records from all of the tables, using the web application.
- The web application should have two types of users/roles - admin, and regular.
- Admin and Regular users have to register and login in order to use the web application. All CRUD (create, read, update and delete) operations, with the exception of DELETE, are available to regular users.
- Admin users should have the ability to perform all CRUD operations on the database tables and make changes to the underlying database data.
- Both regular users and administrators will be able to use the web application.
- Regular users should not be able to make changes to the underlying database data, but administrators should be able to.

Validation should be implemented so that the user cannot conduct an invalid operation or enter data that is not in compliance with the field's requirements. When validation rules are broken, appropriate error messages should be displayed.

Guidelines

- The web application should follow the core design rules of modularisation by breaking down the functionality into relevant components.
- The source code of the web application should incorporate features that will improve the code quality and readability by applying basic programming rules such as appropriate naming conventions, indentation, comments, and refactoring.
- Usability of the web application should be considered. For example, after finishing an activity, the user should be presented an appropriate notification indicating whether their activity was successful or unsuccessful. Also, before exiting the application or deleting a record, a confirmation should be presented to the user.

Task 3 - Report on Agile Overview (50%)

Estimated 1500 words.

Select an aspect of Agile software development approach that is of interest to you and your organization, for example - (i) approach to running IT/software development projects; (ii) approach to running IT/software development teams (including roles and responsibilities of team members); (iii) approach to measuring IT/software development operations and tasks; (iv) approach to automating IT/software development operations and tasks:
- Compare, contrast, and critically evaluate the essential features of your IT/Software development approach within your workplace to an Agile software development framework such as SCRUM, KANBAN, LEAN, etc. Ensure that you cover all the important features of agile project development principles that relates to the framework you have chosen.
- Propose an approach for introducing/improving an agile development approach in a way that would benefit your organisation. Suggest ways of implementing your proposed approach.
